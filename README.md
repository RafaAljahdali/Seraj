# Seraj

Seraj is a clinical monitoring dashboard built as a prototype inspired by NewTech Hackathon.  
It helps doctors quickly identify patient deterioration risk using vitals/labs data and provides a short explanation (rule-based + optional Gemini LLM).

---

## Features
- Dashboard showing patient risk tiers (MINIMAL / LOW / MODERATE / HIGH / CRITICAL)
- Patient details page (timeline for vitals/labs)
- Manual **Predict + Explain** (enter vitals/labs and get risk + explanation)
- Explanation modes:
  - **Local (rule-based)** if no API key
  - **Google Gemini (LLM)** if `GEMINI_API_KEY` is set

---

## Dataset
This project uses the **Hospital Deterioration Dataset** from Kaggle:  
https://www.kaggle.com/datasets/tarekmasryo/hospital-deterioration-dataset

## Requirements

Python 3.10+ (recommended)

## Install Dependencies
```
pip install -r requirements.txt
```

## Google Gemini API Key (Optional)

If you want the explanation to be generated by Gemini:
1) Create your API key: <a href="https://aistudio.google.com/api-keys" target="_blank">Create a Gemini API key</a>
2) Set it before running the app.

## Run the App (Windows CMD)
1) Open CMD in the project folder
2) If you have a Gemini API key:
```
set GEMINI_API_KEY=YOUR_KEY_HERE
```
3) Run:
```
python app.py
```
4) Open in browser:
```
http://127.0.0.1:5000
```

## Notes
- The dataset/ folder is required. The system loads patients/vitals/labs and uses model.joblib.
- If GEMINI_API_KEY is not set, the system will still work using local explanations.
